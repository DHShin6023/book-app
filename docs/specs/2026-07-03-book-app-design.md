# 도서 추천 앱 설계 문서

날짜: 2026-07-03
상태: 사용자 승인 완료

## 개요

사용자가 현재 마음에 드는 단어 3~10개를 입력하면, OpenRouter AI가 그 단어들의 정서/주제에 맞는 실존 도서 3~5권을 추천해주는 웹앱(PWA). 도서 범위는 한국 도서 + 해외 도서. 기존 칼로리앱/관상앱과 동일한 패턴(정적 사이트 + OpenRouter API + GitHub Pages).

## 확정된 요구사항

- **입력**: 단어 3~10개 (3개 미만이면 추천 버튼 비활성, 10개 도달 시 입력 잠금)
- **출력**: 도서 3~5권, 카드 형태
- **카드당 표시 정보**: 표지 이미지, 제목, 저자, 추천 이유(입력 단어와의 연결 설명), 줄거리 2~3문장
- **도서 범위**: 한국 도서 + 해외 도서
- **API 키**: 각 사용자가 설정 화면에서 본인의 OpenRouter 키를 직접 입력, localStorage 저장 (칼로리앱과 동일)
- **표지 이미지**: Google Books API (키 불필요) — AI 환각(없는 책) 검증 역할 겸함

## 화면 구성 (단일 페이지)

1. **메인 화면**
   - 단어 입력창: 입력 후 엔터/추가 버튼으로 칩(태그) 형태로 쌓임, 칩별 삭제(×) 가능
   - 카운터 표시 (예: 4/10)
   - "도서 추천 받기" 버튼: 3개 미만 비활성
2. **결과 영역**
   - 버튼 아래로 책 카드 3~5장 렌더링
   - 로딩 중: "AI가 책을 고르는 중..." 스피너
3. **설정 화면**
   - 톱니 아이콘으로 진입
   - OpenRouter API 키 입력, 모델 선택(기본: 무료 모델)

## 기술 구조

- 순수 HTML/CSS/JS (프레임워크 없음)
- 파일: `index.html`, `style.css`, `app.js`, `manifest.json`, `sw.js`, `icons/`
- PWA: 아이폰 홈 화면 설치 지원
- 저장: localStorage (API 키, 모델 선택)
- 위치: `~/Desktop/ClaudeWork/book-app/`
- 배포: GitHub Pages → `dhshin6023.github.io/book-app`

## 데이터 흐름

1. 단어 3~10개 수집
2. OpenRouter `chat/completions` 호출
   - 프롬프트: 단어들의 정서/주제에 맞는 **실존 도서** 3~5권을 한국+해외 범위에서 추천, 반드시 JSON 배열로만 응답
   - JSON 필드: `title`(한국어판 제목 우선), `author`, `reason`(입력 단어와의 연결), `summary`(줄거리 2~3문장), `original_title`(해외 도서인 경우 원제 — Google Books 검색 보조용)
3. 응답 JSON 파싱
4. 책마다 Google Books API를 **병렬** 호출: `https://www.googleapis.com/books/v1/volumes?q=<제목>+<저자>` → 표지 썸네일 URL 획득
5. 카드 렌더링

## 에러 처리

| 상황 | 처리 |
|---|---|
| API 키 미입력 | "설정에서 API 키를 입력해주세요" 안내 후 설정 화면 이동 |
| AI 응답이 JSON 파싱 실패 | 자동 1회 재시도, 재실패 시 에러 메시지 |
| Google Books에서 표지 못 찾음 | 책 제목이 들어간 기본 표지(placeholder) + "표지 정보 없음" 배지 |
| 네트워크 오류/한도 초과 | 사용자 친화적 한국어 메시지 |

## 테스트 (배포 전 로컬 확인)

- 단어 3개(최소)/10개(최대) 경계값 동작
- 2개 이하에서 버튼 비활성 확인
- API 키 미입력 시 안내 동작
- 실제 API 호출로 추천 결과 및 표지 표시 확인
- 표지 없는 책 placeholder 동작
